## 적용 스킬 핵심 규칙 요약

### folder-structure
- 배치 기준: page는 `pages`, 상호작용은 `features`, 범용은 `shared`
- 금지: 같은 계층 slice 직접 import, 상향 import, index 우회 import
- i18n: 사용자 노출 문자열 하드코딩 금지

### code-quality
- 가독성: 이름만 보고 역할이 드러나야 함
- 예측 가능성: 상태 전이와 부수효과를 분리
- 응집도: 함께 변하는 로직을 같은 모듈에 둠
- 결합도: 구현 세부가 외부에 새지 않도록 경계 설정

### vercel-react-best-practices
- `rerender-memo`, `rendering-hoist-jsx`, `rendering-conditional-render` 우선 적용
- 데이터 연동 구조는 `async-parallel` 고려
- 번들 최적화를 위해 `bundle-barrel-imports`, 필요 시 `bundle-dynamic-imports` 적용
